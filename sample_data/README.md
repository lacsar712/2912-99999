# 生产线监控系统 - 示例数据

本目录包含生产线监控系统的示例数据文件，用于测试数据导入功能和系统演示。

## 文件说明

### 1. `production_lines.csv` - 生产线数据
- 10条生产线示例数据
- 包含生产线编号、名称、描述、状态、产能、位置、负责人等信息
- 状态包括：running（运行中）、stopped（停止）、maintenance（维护中）、error（错误）

### 2. `equipments.csv` - 设备数据
- 17台设备示例数据
- 包含设备编号、名称、类型、所属生产线ID、状态、型号、制造商等信息
- 设备类型包括：注塑机、冲压机、焊接机器人、装配线、包装机等
- 状态包括：running（运行中）、idle（空闲）、maintenance（维护中）、error（错误）、offline（离线）

### 3. `sensors.csv` - 传感器数据
- 28个传感器示例数据
- 包含传感器编号、名称、类型、所属设备ID、单位、阈值、状态、最后读数等信息
- 传感器类型包括：temperature（温度）、pressure（压力）、humidity（湿度）、speed（速度）、vibration（振动）、current（电流）、voltage（电压）

### 4. `tasks.csv` - 生产任务数据
- 12个生产任务示例数据
- 包含任务编号、名称、生产线ID、产品名称、规格、计划数量、已完成数量、状态、优先级、时间等信息
- 状态包括：pending（待开始）、in_progress（进行中）、paused（暂停）、completed（已完成）、cancelled（已取消）

### 5. `records.json` - 生产记录数据
- 7条生产记录示例数据（JSON格式）
- 包含设备ID、任务ID、生产数量、合格数量、缺陷数量、良品率、温度、湿度、运行时长、生产效率、记录时间等信息

### 6. `alerts.json` - 告警记录数据
- 7条告警记录示例数据（JSON格式）
- 包含告警编号、类型、设备ID、传感器ID、严重程度、消息、告警值、阈值、状态、解决时间、解决备注等信息

## 数据导入方法

### 方法一：通过Web界面导入
1. 登录生产线监控系统
2. 进入"数据模拟"页面
3. 点击"上传文件"按钮
4. 选择对应的CSV或JSON文件
5. 确认导入设置
6. 等待导入完成

### 方法二：通过API批量导入
```bash
# 导入生产线数据
curl -X POST "http://localhost:5001/api/simulation/import/csv" \
  -H "Content-Type: multipart/form-data" \
  -H "Authorization: Bearer <your-token>" \
  -F "file=@production_lines.csv" \
  -F "type=production_line"

# 导入JSON数据
curl -X POST "http://localhost:5001/api/simulation/import/json" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d "@records.json"
```

### 方法三：使用模拟数据生成功能
1. 进入"数据模拟"页面
2. 配置模拟参数（生产线数量、设备数量等）
3. 点击"生成模拟数据"按钮
4. 系统自动生成符合配置的模拟数据

## 数据验证

系统提供数据验证功能，确保导入的数据符合要求：

### 1. 完整性检查
- 必填字段验证
- 数据格式验证
- 关联关系验证（如设备必须属于有效的生产线）

### 2. 业务规则验证
- 状态值必须在允许范围内
- 数值范围验证（如温度必须在合理范围内）
- 唯一性验证（如生产线编号不能重复）

### 3. 关联性验证
- 外键关系验证
- 数据一致性验证
- 引用完整性验证

## 高级功能

系统提供数据验证功能，确保导入的数据符合业务规则：

```javascript
// 前端验证示例
const validationResult = await SimulationService.validateData('production_line', dataArray);
```

### 2. 模拟数据源

#### API数据源
- 模拟REST API数据输入
- 可配置端点、方法和间隔

#### WebSocket数据源
- 模拟实时数据流
- 支持双向通信模拟

#### 文件流数据源
- 模拟文件持续输入
- 支持CSV、JSON等格式

#### 用户输入数据源
- 模拟用户表单输入
- 可配置输入字段和验证规则

```javascript
// 创建API数据源
const sourceConfig = {

    method: 'GET',
    interval: 5

const sourceResult = await SimulationService.createMockDataSource('api', sourceConfig);

// 从数据源生成数据
const generateResult = await SimulationService.generateFromSource(sourceResult.data.source_id, 10);
```

## 性能优化建议

### 大规模数据导入
1. **分批导入**：超过1000条记录时，建议分批导入
2. **索引优化**：确保数据库表有适当索引
3. **事务管理**：使用事务保证数据一致性

### 数据生成优化
1. **调整配置**：根据需求调整数据规模和分布
2. **缓存策略**：重复生成时考虑缓存机制
3. **异步处理**：大规模生成使用异步任务

## 故障排除

### 常见问题及解决方案

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 导入失败 | 文件格式错误 | 检查CSV列名或JSON结构 |
| 数据不完整 | 必填字段缺失 | 验证数据完整性 |
| 性能缓慢 | 数据量过大 | 分批导入，优化查询 |
| 关联错误 | 外键不存在 | 确保关联数据已导入 |

### 调试方法
1. 查看浏览器控制台日志
2. 检查后端服务日志
3. 使用数据验证功能
4. 分步测试导入流程
